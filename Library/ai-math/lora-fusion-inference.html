LoRA Fusion for Inference

Low-Rank Adaptation (LoRA) is the standard for efficient fine-tuning. While adapters are efficient to train, they add latency during inference if executed as separate layers. Symbolic fusion allows us to merge the adapter weights directly into the base model, achieving "zero-overhead" inference.

The Problem

The Low-Rank Latency Penalty

A LoRA layer modifies a base weight W using two low-rank matrices A and B. During inference, the output is calculated as:
Y = X × W + (X × A) × B

Executing this as written requires two additional matrix multiplications (one for A and one for B) for every layer. For high-throughput serving, this additional compute can reduce total tokens-per-second.

Symbolic Insight

Pre-computing the Delta

Using the distributive property of matrix multiplication, a symbolic engine can refactor the expression:
X × W + X × (A × B) = X × (W + A × B)

This transformation enables **Weight-Space Fusion**:

  • Static Merge: Before the model starts serving, compute Wfused = W + A × B.
  • Zero Overhead: During inference, simply compute Y = X × Wfused.

The resulting Wfused has the same shape as W, meaning the adapter now consumes zero extra FLOPs at runtime.

Why It Matters

Multi-Tenant LLM Serving

In "LoRA-Exchange" systems (like vLLM or LoRAX), symbolic fusion is the key to serving hundreds of different fine-tuned models on the same GPU base.

While dynamic switching between adapters requires special kernels (like SGMV), static fusion remains the gold standard for dedicated model instances, ensuring no performance regression compared to the base model.
