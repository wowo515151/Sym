Multi-Query Attention (MQA) Optimization

Multi-Query Attention (MQA) reduces the memory footprint of transformer models by sharing Key and Value heads across multiple Query heads. Symbolic re-association can further optimize the computational cost of these shared projections.

The Problem

Asymmetric Shapes in Attention Projections

In Multi-Query Attention, the Key (K) and Value (V) tensors have significantly fewer dimensions than the Query (Q) tensor along the head axis. This creates an asymmetry in the matrix multiplication chain: (Q × KT) × V.

When the context length (the number of rows in K and V) is large, but the number of heads is small, the intermediate product Q × KT can be much larger than the input or output tensors.

Symbolic Insight

Preferring Context-Value Pre-computation

Consider MQA shapes:
Q = [128, 64]
K = [1024, 64]
V = [1024, 128]

  • Standard Order: (Q × KT) × V computes a [128, 1024] attention matrix.
  • Optimized Order: Q × (KT × V) computes a [64, 128] summary matrix.

By re-associating to Q × (KT × V), the system avoids materializing the large sequence-by-sequence attention matrix, leading to dramatic memory savings and faster execution for long context windows.

Proof

Symbolic Trace

// Original MQA Chain MatMul(MatMul(Q, Transpose(K)), V)
