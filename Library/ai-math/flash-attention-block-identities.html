Flash-Attention Block Identities

Standard Attention is quadratic in sequence length, creating a memory bottleneck. Flash-Attention uses symbolic tiling and recomputation to break the large attention matrix into blocks that fit in GPU SRAM, eliminating the HBM bottleneck entirely.

The Problem

The HBM Memory Wall

In standard attention, we compute S = Softmax(QKT) and then O = SV. For a sequence of 8k tokens, the matrix S is 8000 × 8000, which is 64 million elements.

Writing this massive matrix to the GPU's main memory (HBM) and reading it back is the primary bottleneck. Most of the GPU's compute power sits idle while waiting for these memory transfers.

Symbolic Insight

Tiling and the Softmax Decomposition

The breakthrough of Flash-Attention is the symbolic realization that Softmax(QKT) can be calculated incrementally. By tiling Q, K, and V into small blocks (e.g., 128 × 128), we can compute the partial attention for that block and store it in high-speed **SRAM**.

This depends on the **Online Softmax Identity**:
Softmax(Xtotal) = Update(Softmax(Xblock1), Softmax(Xblock2))

By maintaining a running normalization factor, we can compute the entire Attention mechanism without ever materializing the full N × N matrix in HBM.

Why It Matters

The Road to 1M Token Context

Flash-Attention (and its successor Flash-Attention-2) is why we now have LLMs with 128k, 1M, or even larger context windows. It changes the fundamental compute profile from **memory-bound** to **compute-bound**.

Symbolic engines facilitate this by identifying the high-level Attention(Q, K, V) operator and automatically mapping it to the tiled implementation, ensuring that the complex re-scaling logic is mathematically verified.
