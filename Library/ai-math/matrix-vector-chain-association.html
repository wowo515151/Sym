Matrix-Vector Chain Association

The order in which we multiply a chain of matrices can change the algorithmic complexity of the operation. In the case of matrix-vector products, the difference is often between a cubic O(N^3) operation and a quadratic O(N^2) one.

The Problem

The O(N^3) Trap

Consider the expression (A × B) × x, where A and B are large matrices and x is a vector.

  • If we compute (A × B) first, we perform a matrix-matrix product. This is an O(N^3) operation.
  • The resulting matrix is then multiplied by the vector x, which is O(N^2).
Symbolic Insight

Re-associating for O(N^2) Complexity

If we re-associate the expression to A × (B × x), the complexity profile changes completely:

  • First, compute the matrix-vector product B × x (an O(N^2) operation).
  • Then, multiply the matrix A by the resulting vector (another O(N^2) operation).

The total complexity drops from O(N^3) to O(N^2). For a matrix of size 1000x1000, this is a 1000-fold reduction in compute effort.
