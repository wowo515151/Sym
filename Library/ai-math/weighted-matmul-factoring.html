Shared Weight Factoring and Stacking

In many AI architectures, multiple distinct input tensors are multiplied by the same weight matrix. Symbolic optimization can identify opportunities to "stack" these operations into a single large matrix multiplication, improving GPU utilization and memory bandwidth efficiency.

The Problem

Redundant Weight Loads in Parallel Paths

Consider a scenario where two inputs X and Y are processed by the same weights W, each with a different scaling factor g1 and g2:
Result = (X × W) × g1 + (Y × W) × g2

A naive implementation executes two separate MatMul kernels. Because W is shared, this results in loading the large weight matrix from memory twice, which is a significant bottleneck on hardware that is memory-bandwidth limited.

Symbolic Insight

Factoring and Stacking

While basic factoring would transform the expression to (X × g1 + Y × g2) × W, the Sym engine identifies an even more efficient form for modern hardware: Stacking.

  • Original: Two separate kernels, two loads of W.
  • Factored: One larger addition, one load of W.
  • Stacked: One unified kernel where X and Y are concatenated along a new batch dimension.

For shapes X, Y = [32, 128] and W = [128, 256], stacking allows the GPU to process both inputs in a single kernel launch, maximizing occupancy.

Why It Matters

Memory Bandwidth and Kernel Fusion

Modern GPUs have massive compute power but limited memory bandwidth. Loading a large weight matrix is expensive. By stacking the inputs, we ensure that the weight matrix W is read into the GPU's registers once and reused across all inputs in the stack.

This is a foundational optimization for Mixture of Experts (MoE) and Multi-Query Attention (MQA), where shared weights are a primary architectural feature.

// Original Expression TensorAdd(TensorMul(MatMul(X, W), g1), TensorMul(MatMul(Y, W), g2))

// Optimization Rule Applied: StackWeightedSum TensorAdd(TensorMul(?r1, ?w1), TensorMul(?r2, ?w2)) -> MatMul(Stack(0, ?w1, ?w2), Stack(0, ?r1, ?r2))

// Result (Verified for Shapes) MatMul(MatMul(Stack(0, g1, g2), Stack(0, X, Y)), W)
