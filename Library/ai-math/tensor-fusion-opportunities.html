Tensor Fusion: Bias and Activation

One of the most effective ways to optimize neural network performance is kernel fusion. Symbolic engines can detect patterns like Relu(A × B + C) and replace them with highly optimized, unified kernels.

The Problem

The Cost of Kernel Overhead

In a typical deep learning layer, we perform a Matrix Multiplication, followed by a Bias Addition, followed by a Non-linear Activation (like Relu). If executed separately, this requires three distinct GPU kernel launches and multiple round-trips to global memory.

Symbolic Insight

Pattern Matching for Fusion

A symbolic system can represent the entire sequence as a single expression:
Relu(TensorAdd(MatMul(A, B), C))

Instead of optimizing each operation in isolation, the Sym engine looks for Fusion Targets. By identifying the FusedMatMulAddRelu pattern, the system instructs the compiler to use a single, hand-optimized CUDA or Triton kernel that performs all three steps in one pass through the GPU's registers.
