Online Softmax Fusion

Standard Softmax is a "three-pass" algorithm: find max, compute sum of exps, then normalize. Symbolic re-formulation into the "Online Softmax" allows these three steps to be fused into a single GPU kernel pass, dramatically reducing memory bandwidth and latency.

The Problem

The Three-Pass Memory Wall

A naive Softmax implementation on a GPU requires reading the input tensor from global memory three separate times:

  1. Pass 1 (Max): Find m = max(xi) for numerical stability.
  2. Pass 2 (Sum): Compute S = Σ exp(xi - m).
  3. Pass 3 (Norm): Compute yi = exp(xi - m) / S.

For a large attention matrix, the time spent moving data between HBM and the GPU registers dominates the actual exponential math.

Symbolic Insight

Recurrent Normalization

Symbolically, we can represent the max and sum as recurrent updates. If we have a partial max mold and a partial sum Sold, and we encounter a new element xnew:

  • New Max: mnew = max(mold, xnew)
  • Rescaled Sum: Snew = Sold × exp(mold - mnew) + exp(xnew - mnew)

This recurrence allows the GPU to compute the final stable softmax result while streaming through the data **exactly once**. The symbolic engine identifies the Softmax(X) operator and replaces the three-pass DAG with a single OnlineSoftmax kernel.

Why It Matters

Throughput in Long-Context Attention

Online Softmax is the secret sauce behind Flash-Attention. By enabling single-pass execution, it allows the attention mechanism to stay entirely within the GPU's fast SRAM (L1 cache) without ever writing intermediate results back to slower main memory.

This reduces the memory bandwidth requirement of the softmax layer by **66%**, which is directly proportional to the speedup seen in modern LLM training and inference.
