KV Cache Quantization Folding

The KV cache is the largest memory consumer in LLM serving. Quantizing these caches to 4-bit (INT4) or 8-bit (FP8) is essential for large context windows. Symbolic scale folding allows the quantization "scales" to be pushed through the attention math, minimizing precision loss.

The Problem

The Storage vs. Precision Conflict

A 128k context window in a large model can consume 50GB+ of HBM. If we quantize the Key (K) and Value (V) tensors to 4-bit, we save 75% of that memory. However, every time we perform an Attention MatMul, we must "dequantize" the values back to FP16 to perform the math.

Naively dequantizing inside the hot-loop adds a massive number of instructions and can lead to rounding errors that accumulate across heads.

Symbolic Insight

Scale Propagation

A quantized tensor is represented as Q(X) = X_int / Scale. When calculating attention (Q × KT), we can pull the scales out:
(Q × K_intT / S_k) = (Q × K_intT) / S_k

Symbolic engines perform **Scale Propagation**:

  • Fold to Scalar: Instead of dequantizing every element of K, the engine performs the MatMul in INT-space and applies the scalar scale 1/S_k to the *result*.
  • Hoisting: The scales for K and V are hoisted out of the reduction loop and combined with the Softmax scaling factor.

This allows the GPU to stay in high-speed integer mode for the bulk of the work, only returning to floating-point at the very end of the attention block.

Why It Matters

Beyond the 80GB Limit

Scale-aware quantization is what makes it possible to serve an 800B parameter MoE model on a single node. By symbolically managing the scales, systems like KIVI or AutoFP8 can maintain near-FP16 accuracy while using 4-bit storage.

This optimization is the frontier of "Infinite Context" AI, where the goal is to fit as much history as possible into the GPU's fixed memory budget.
