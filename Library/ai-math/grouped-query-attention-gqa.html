Grouped-Query Attention (GQA)

Grouped-Query Attention (GQA) is the architectural breakthrough that allows models like Llama-3 and Mistral to maintain high quality while significantly reducing memory bandwidth pressure. Symbolically, GQA is a generalized form of attention that groups multiple query heads to share a single key-value pair.

The Problem

The Memory Wall: MHA vs. MQA

In Multi-Head Attention (MHA), every query head has its own KV head, leading to massive KV caches that don't fit in GPU memory for long sequences. In Multi-Query Attention (MQA), all query heads share one KV head, which saves memory but can degrade model quality.

GQA solves this by partitioning query heads into G groups. Each group shares one KV head. This is the "Goldilocks" solution for modern LLMs.

Symbolic Insight

Representing Head Partitioning

Symbolically, GQA is represented as a Blocked Reshape and Repeat operation on the KV tensors before the attention MatMul.

Instead of a 1:1 or N:1 mapping, the engine sees a GroupedRepeat pattern:
K_grouped = Repeat(K, groups=H/G)

This allows the symbolic optimizer to identify Broadcast-MatMul fusion opportunities. Instead of physically repeating the K/V tensors in memory (which wastes space), the engine instructs the GPU to use a broadcast-aware attention kernel that reads each KV head once and uses it for all queries in the group.

Why It Matters

Maximizing Tokens-Per-Second

The primary bottleneck for LLM inference is Memory Bandwidth, not compute. GQA reduces the amount of KV data that must be loaded from HBM (High Bandwidth Memory) for every token generated.

By symbolically optimizing the GQA dispatch, we can ensure that the GPU's Tensor Cores stay busy while the memory controllers handle a much smaller data stream, resulting in 2x-3x higher inference throughput for large-batch serving.
