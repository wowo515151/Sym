# SymCLI Skill Help

This skill wraps SymCLI through `symcli.bat` on Windows or `symcli.sh` on Unix-like systems.

The wrappers are repo-relative. They look for built SymCLI outputs under `src/SymCLI/bin/Release/net10.0` and `src/SymCLI/bin/Debug/net10.0`, then fall back to `dotnet run --project src/SymCLI/SymCLI.csproj` if no built executable is present.

SymCLI is a practical AI math tool because it is just a console app with a help file, predictable file inputs, and predictable file outputs. That makes it easy to use in Skills-style agent workflows, and also easy to wrap behind richer agent/tool protocols when needed.

## What It Does

SymCLI currently supports two main workflows:

1. Solving and optimizing ProblemScript (`.ps`) inputs with the Sym E-Graph engine.
2. Analyzing C# source code for mathematical and security-oriented bug patterns.

## Usage

### Solve / optimize ProblemScript

```text
symcli.bat <inputPath.ps> <outputPath>
symcli.sh <inputPath.ps> <outputPath>
```

- `inputPath.ps`: ProblemScript input
- `outputPath`: output file for the result

### Analyze C# for math / security issues

```text
symcli.bat analyze csharp-math <inputPath> <outputPath> [options]
symcli.sh analyze csharp-math <inputPath> <outputPath> [options]
```

- `inputPath`: a single `.cs` file or a directory to scan recursively
- `outputPath`: text or JSON report path

To print analysis help:

```text
symcli.bat analyze csharp-math --help
symcli.sh analyze csharp-math --help
```

## ProblemScript Notes

ProblemScript can contain:

- an `<Options>...</Options>` block
- equations or constraints
- script-defined rewrite rules

Example:

```xml
<Options>
  Target: x
  RulePacks: Algebraic,Integration
  CostModel: Default
  MaxIterations: 1000
  MaxConcurrency: 8
  EnableTracing: false
</Options>
```

Useful option fields include:

- `Target`
- `RulePacks`
- `CostModel`
- `MaxIterations`
- `MaxConcurrency`
- `EnableTracing`
- `Domain`
- `AssumePositive`
- `AssumeReal`
- `AssumeComplex`
- `AssumeInteger`
- `TreatNonEqualitiesAsError`
- `Shapes`
- `Attributes`

Example constraint:

```text
x^2 + 2*x + 1 = 0
```

Example custom rule:

```text
Rule(a + a, 2 * a);
```

## Analyze Mode

The C# analyzer emits findings such as:

- `CSMATH...` for mathematical correctness hazards
- `CSSEC...` for security-oriented patterns

This is useful when a coding agent needs an exact analyzer for numeric code, symbolic code, or AI-generated C# rather than relying on informal inspection alone.

### Options

These match the CLI’s built-in help:

- `--stability`
- `--max-findings <int>`
- `--confidence-threshold <f>`
- `--max-iterations <int>`
- `--sample-budget <int>`
- `--saturation-timeout-seconds <f>`
- `--analysis-timeout-seconds <f>`
- `--max-parallelism <int>`
- `--security-flow-mode <mode>`
- `--security-max-trace-steps <int>`
- `--user-source-only`
- `--external-source-only`
- `--repo-scan-info <path>`
- `--disable-guard-prover`
- `--guard-max-facts <int>`
- `--guard-max-iterations <int>`
- `--guard-timeout-seconds <f>`
- `--json`
- `--include-security`
- `--math-only`
- `--fail-on-findings`

### Directory Scan Behavior

When given a folder, SymCLI:

- recursively searches for `.cs` files
- skips common build/artifact folders such as `bin`, `obj`, `.git`, `.vs`, `.vscode`, `node_modules`, `packages`, and test/example folders
- auto-loads `RepoScanInfo.txt` from the input directory when present unless `--repo-scan-info` is supplied explicitly

## Exit Codes

### Solve mode

- `0`: success
- `1`: configuration or input error
- `2`: solving failed; diagnostics were written
- `3`: unexpected runtime exception

### Analyze mode

- `0`: analysis completed successfully
- `1`: argument, option, or input-path error
- `3`: unexpected runtime exception
- `4`: findings were present and `--fail-on-findings` was supplied
- `5`: cancelled or timed out

## AI Workflow Note

SymCLI is a good fit for an "AI mathematician" workflow:

- the agent interprets the mathematical or coding task
- SymCLI performs exact symbolic or code-analysis work
- the agent reads the result and continues with stronger grounding

That makes symbolic computation complementary to AI. The agent provides flexibility and planning; SymCLI provides exact mathematical execution and analysis.
