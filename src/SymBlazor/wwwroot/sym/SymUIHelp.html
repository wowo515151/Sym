Sym UI Help

This page replaces the old in-app popup help. Open it in a separate tab while using the Blazor UI so you can keep the app visible and still read the instructions.

Open The App Open the full Sym reference

Toolbar

  • Start: Runs the current ProblemScript through Sym's EGraph wrapper.
  • Stop: Cancels the current solve request.
  • Examples: Loads a predefined example script into the input area.
  • Help: Opens this page in a new tab.

Input And Output

  • The Input pane accepts ProblemScript text.
  • The Output pane shows the formatted solve result or an error message.
  • The UI does not try to simplify partial fragments automatically; press Start to run the engine.

What To Type

The UI accepts the same ProblemScript format used by other Sym wrappers. You can provide plain expressions, equations, an <Options> block, and an optional <Rules> block for inline rewrite rules. 

<Options>
  RulePacks: Tensor, EquationSolving
  CostModel: Tensor
  Target: x
</Options>

x + 5 = 10

Common Options

  • Target tells Sym which symbol to solve for.
  • RulePacks narrows the folder-based rule packs to use.
  • MaxIterations limits EGraph saturation work.
  • EnableTracing enables richer internal trace output.

Tensor Work

  • RulePacks: Tensor enables tensor and fused-operator rules.
  • CostModel: Tensor helps extraction prefer cheaper tensor forms.
  • Shapes can be supplied for symbols such as A=[64,128].

Inline Rules

You can add custom rewrite rules directly in a <Rules> block. Those rules are combined with the standard Sym rule libraries. 

<Rules>
  Rule(f(Wild("x")), Pow(Wild("x"), 2));
</Rules>

f(5) = y

When The Result Looks Wrong

  • Check for parse errors in the output pane.
  • Try specifying Target explicitly for equation solving.
  • Reduce the selected RulePacks if too many unrelated rewrites are being considered.
  • For tensor problems, add Shapes and a tensor cost model.
